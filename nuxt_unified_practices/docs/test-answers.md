# Шпаргалка по тестам из чата

## Nuxt UI / переменные

- Массив, который можно дополнять: `let array = []` и `const array = []`.
- Перезаписываемая переменная: `let`.
- Аватар пользователя: `UAvatar`.
- Выпадающий выбор: `USelectMenu`.
- Связь значения с переменной: `v-model`.
- Радиоменю: `URadioGroup`.
- Карточка: `UCard`.
- Прогресс-бар: `UProgress`.
- Глобальные стили: `nuxt.config.ts` через `css`.
- Маршруты Nuxt создаёт по файлам в `pages/`.

## Практика 1

- Добавить нового блогера в MockAPI: `POST`.
- Получить данные без изменения сервера: `GET`.
- Асинхронные данные: `useAsyncData`.
- Список по массиву: `v-for`.
- Параметры в `useFetch`: второй аргумент `{ query: { key: value } }`.
- `randomuser.me` на 50 пользователей: `https://randomuser.me/api/?results=50`.
- По тесту обязательные поля при создании: `id, description`.

## Практика 2

- `/index` для SEO и скорости: `prerender: true`.
- `/user_page`: `swr: true`.
- `/recommendations`: `swr: 3600`.
- `/admin` CSR: `ssr: false`.
- `/login` → `/enter`: `redirect`.

## Практика 3

- `useState` создаёт реактивную переменную для хранения данных.
- Получение данных с API по тесту: `useAsyncData`.
- `useRoute` возвращает объект текущего маршрута.
- `defineNuxtComponent` определяет компонент Nuxt.
- Заголовок страницы: `useHead`.
- Реактивный выбор гигабайт: `useState`.
- SSR-состояние вместо `ref`: `useState`.
- Первый аргумент `useState`: строковый ключ состояния.
- Основная проблема `ref` при SSR: ошибки гидратации.
- Нереактивные функции: `utils/`.
- Composables называются с префиксом `use`.
- Выбор языка по тесту: `useLanguage`.
- Composables нужны, если нужна реактивность и хранение состояния.
